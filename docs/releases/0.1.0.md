# Release Note

## 新功能

forgectl 的首個版本, 提供透過 GitHub / GitLab REST API 查詢與操作 repo release 與
asset 的完整 CLI. 本版的全部功能如下:

- **跨平台支援 (GitHub / GitLab)**: 以 `--source github|gitlab` 指定平台, 涵蓋公開站
  (github.com / gitlab.com) 與自架站 (GitHub Enterprise / 自架 GitLab); 自架站以
  `--host` 指定 base URL, api_base 依慣例自動推導. 各指令一律走同一套跨平台邏輯.

- **`ping` — 驗證連線與認證設定**: 對連線目標做分層檢查 — 先以不需 token 的請求驗證
  連線 / `--host` / TLS, 再 (有 token 時) 對 current-user 端點驗證認證並回報身分;
  匿名為合法用法. 只印 token 來源, 絕不印 token 本體.

- **`release list` — 列出 repo 的所有 release**: 預設輸出供人閱讀的文字格式, 亦可 `--json`
  供腳本使用. 標示 draft / prerelease / upcoming 狀態, 顯示每筆 release 的 commit hash
  與使用者上傳的 asset 清單 (排除自動產生的原始碼壓縮檔).

- **`release create` — 發布某 version 的正式 release**: 將該 version 已上傳的 asset 一併
  連上發布. 以 `--note` 或 `--note-file` 提供 release note; tag 不存在時以 `--commit`
  (commit SHA 或 `latest`) 明確指定指向的 commit. 目標 version 已是正式 release 時報錯.

- **`asset upload` — 上傳本地檔為某 version 的 asset**: 一次可上傳一個以上的本地檔,
  release 不需先存在. 支援 `<path>=NAME` 改名, 同名既有 asset 一律覆蓋; 多檔逐一全部
  嘗試, 結束時彙總成功與失敗清單.

- **`asset download` — 下載某 release 的 asset**: 以 glob 選取 asset 名 (省略則全部),
  version 可用 `latest` 取最新正式 release. 支援 `-d` 指定目錄, `-o` 指定輸出檔名,
  `--overwrite` 覆寫既有檔; pattern 無命中時不視為錯誤.

- **以 credential 檔為基底的認證**: 認證以固定位置的 credential 檔 (TOML / YAML / JSON)
  為單一事實來源, 旗標 `--token` / `--token-file` / `--user` 僅做欄位層級覆寫. credential
  檔權限可被他人讀取時印警告至 stderr.

- **TLS 憑證驗證控制 (`--insecure`)**: 全域旗標, 供自架 host 使用自簽憑證時跳過 TLS
  憑證驗證; 使用時印警告至 stderr.
